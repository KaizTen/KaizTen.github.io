    Introduction

    REST API used manage information about users and their services, employees, and base schedules to determine the sequence of services to do by each employee.

    The API supports Cross Origin Resource Sharing (CORS) for AJAX requests from any origin. The following header is included in the corresponding responses: Access-Control-Allow-Origin: *. You can read the CORS W3C Recommendation to learn more about CORS.

    In the remainder of this documentation, it is assumed that the REST API is deployed in an IP and port denoted as IP and PORT, respectively.

    API versioning

    Smart SAD API is versioned with the version specified on the URL route: IP:PORT/VERSION/API_ENDPOINT

    The current supported API version is v1 (i.e. IP:PORT/v1/API_ENDPOINT).

    Domain model

    Employee

    • id (generated). Identifier of the employee.
    • name. Name of the employee.
      • Its length must be in the range [2..50].
      • Only alphabetic characters are allowed. Accents in the characters and ª can be also used.
    • surname. Surname of the employee.
      • Its length must be in the range [2..100].
      • Only alphabetic characters are allowed. Accents in the characters can be also used.
    • code. High level identifier of the employee used by the company. It is a number composed of at most 5 digits.
    • identificationNumber. Identification number of the employee. It can be a DNI, NIF, NIE, or CIF.
    • sex (optional). Sex of the employee. It can be MAN or WOMAN.
    • birthDate (optional). Birth date of the employee. It must be expressed in ISO 8601 format (e.g., 1986-03-17).
    • civilStatus (optional). Civil status of the employee. It can be SINGLE, MARRIED, DIVORCED, WIDOWER, or UNKNOWN (default value).
    • comment (optional). Free comment associated with the employee. Its maximum length is of 1000 characters.
    • address(optional). Address of the employee. It is composed of the following elements:
      • street. Street of the address. Its length must be in the range [2..100].
      • postalCode. Postal code of the address. It is a number composed of 5 digits`.
      • location. Location of the address. Its length must be in the range [2..100].
      • latitude. Latitude of the address.
      • longitude. Latitude of the address.
    • emails (optional). List of emails used to contact the employee with. The length of each email must be in the range [5..100].
    • phones (optional). List of phones used to contact the employee with. Each phone is composed of the following fields:
      • number. Phone number. It is a number composed of 9 digits`.
      • comment. Comment associated with the phone. Its maximum length is of 1000 characters.

    User

    • id (generated). Identifier of the user.
    • name. Name of the user.
      • Its length must be in the range [2..50].
      • Only alphabetic characters are allowed.
    • surname. Surname of the user.
      • Its length must be in the range [2..100].
      • Only alphabetic characters are allowed. Accents in the characters can be also used.
    • code. High level identifier of the user used by the company. It is a number composed of at most 5 digits.
    • identificationNumber (optional). Identification number of the user. It can be a DNI, NIF, NIE, or CIF.
    • sex. Sex of the user. It can be MAN or WOMAN.
    • birthDate (optional). Birth date of the user. It must be expressed in ISO 8601 format (e.g., 1957-09-21).
    • civilStatus (optional). Civil status of the user. It can be SINGLE, MARRIED, DIVORCED, WIDOWER, or UNKNOWN (default value).
    • comment (optional). Free comment associated with the user. Its maximum length is of 1000 characters.
    • address. Address of the user. It is composed of the following elements:
      • street. Street of the address. Its length must be in the range [2..100].
      • postalCode. Postal code of the address. It is a number composed of 5 digits`.
      • location. Location of the address. Its length must be in the range [2..100].
      • latitude. Latitude of the address.
      • longitude. Latitude of the address.
    • emails (optional). List of emails used to contact the employee with. The length of each email must be in the range [5..100].
    • phones (optional). List of phones used to contact the employee with. Each phone is composed of the following fields:
      • number. Phone number. It is a number composed of 9 digits.
      • comment. Comment associated with the phone. Its maximum length is of 1000 characters.
    • services. List of services required by the user. Each service is composed of the following elements:
      • description (optional). Description of the service to do.
      • day. Day in which the service must be do. It can be MONDAY, TUESDAY,..., SUNDAY.
      • minutes (optional). Duration time of the service, expressed in minutes.
      • startingTime. Starting time of the service, expressed as HH:MM:SS.
      • finishingTime. Finishing time of the service, expressed as HH:MM:SS.

    Time

    • id. Identifier of the relationship.
    • code1. Code of the first user. It is a number composed of at most 5 digits.
    • code2. Code of the first user. It is a number composed of at most 5s digits.
    • value. Time between the users, expressed in seconds.
    • mean. Mean used to move between the users. It can be FOOT (default value), CAR, BIKE, or MOTORBIKE.
    • type. Origin of the time. It can be GOOGLE_MAPS, GRAPH-HOPPER, PERSONALIZED, or DEFAULT (default value).
    • lastUpdate (optional). Last time in which the time was updated. It must be expressed in ISO 8601 format (e.g., 2020-12-17T12:57:19+00:00).

    Algorithm

    • id (generated). Identifier of the algorithm.
    • name. Name of the algorithm.
      • Its length must be in the range [2..50].
      • - and _ are valid characters.
      • No whitespaces are allowed.
    • description (optional). Description of the algorithm. Its maximum length is 1000 characters.

    Task

    • id. Identifier of the task.
    • algorithmName. Name of the algorithm associated with the task.
    • status. Status of the task when requested. See Task States.
    • exitCode. Status code to denote the termination of the task. The zero status is used when the execution is terminated successfully. However, a non-zero status code indicates an abnormal termination.
    • output. Output of the task.

    Task states

    • Pending. La tarea aún no ha sido asignada a ningún worker para su realización.
    • Running. La tarea se está realizando.
    • Canceled. La tarea ha sido cancelada.
    • CompletedErrors. La tarea ha sido finalizada porque se han encontrado errores en la ejecución.
    • CompletedTimeout. La tarea ha sido finalizada porque se ha alcanzado el tiempo máximo de ejecución.
    • Complete. La tarea ha sido finalizada con éxito.

    In all the cases, the resource identifiers are generated by the system. This means that the client is not allowed to determine them.

    HTTP request

    GET http://IP:PORT/VERSION/employees

    Query parameters

    Parameter Description
    name Name of the employee to find.
    code Code of the employee to find.
    page Zero-based page index. Default value: 0.
    size Size of the page to be returned. Default value: 20.

    Responses

    HTTP Code Description
    200 The request has succeeded and the employees are included in the body of the response. The following headers are included in the response:
    • Pagination-Count: 1
    • Pagination-Size: 25
    • Pagination-Page: 0
    • Total-Count: 5
    where Pagination-Count is the number of existing pages, Pagination-Size is the size of each page, Pagination-Page is the index of the current page, and Total-Count is the number of entities in the server.
    404 The employee with the requested name is not found.

    Execute an Algorithm

    curl -X POST http://IP:PORT/VERSION/algorithms/:name 
    -H "Content-Type: application/json" \
    -d '{"
            __taskName": "Schedule of users located in SC", 
            "days": 6, 
            "restingTime": "8",
            ...
        }'

    The above command returns JSON structured like this:

    {
   "algorithmId" : "5e70c8aca89d8d39ea0b0420",
   "algorithmName" : "scheduling",
   "taskId" : "142",
   "taskName" : "Schedule of users located in SC",
   "startingTime" : "2020-10-20 12:24:12.095",
   "parameters" : {
      "days" : "6",
      "restingTime" : "8",
      ...
   },
}

    It executes a specific algorithm by indicating its name. The algorithm can receive parameter values in the body of the request in the following formats:

    • Content-Type: text/plain. The parameter values are provided as key-value pairs in text format (UTF-8 encoded). The parameters must be organized by using , whereas = is used to differentiate the parameter from its parameter value.
    • Content-Type: application/json. JSON where each key is the name of the parameter and each value is the corresponding parameter value.

    The following, parameter values can be included in the body of the request with specific purposes:

    • __taskName. It is the name of the execution requested. It should be noted that the response by the API provides a parameter taskName with this value.

    The information about the task created is provided in the response.

    HTTP Request

    POST IP:PORT/VERSION/algorithms/:name

    URL Parameters

    Parameter Description
    name Name of the algorithm to execute.

    Responses

    HTTP Code Description
    202 The request has succeeded and the execution of the algorithm is queued. The information about the task created is provided in the response.
    404 The algorithm with the given name is not found.
