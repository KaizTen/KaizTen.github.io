    Introduction

    REST API used manage information about users and their services, employees, and base schedules to determine the sequence of services to do by each employee.

    The API supports Cross Origin Resource Sharing (CORS) for AJAX requests from any origin. The following header is included in the corresponding responses: Access-Control-Allow-Origin: *. You can read the CORS W3C Recommendation to learn more about CORS.

    In the remainder of this documentation, it is assumed that the REST API is deployed in an IP and port denoted as IP and PORT, respectively.

    API versioning

    Smart SAD API is versioned with the version specified on the URL route: IP:PORT/VERSION/API_ENDPOINT

    The current supported API version is v1 (i.e. IP:PORT/v1/API_ENDPOINT).

    Domain model

    Employee

    • id (generated). Identifier of the employee.
    • name (required). Name of the employee.
      • Its length must be in the range [2..50].
      • Only alphabetic characters are allowed.
    • surname. Surname of the employee.
    • code. High level identifier of the employee used by the company. It is a number composed of 4 digits.
    • sex. Sex of the employee. It can be MAN or WOMAN.
    • birthDate (optional). Birth date of the employee.
    • civilStatus (optional). Civil status of the employee. It can be SINGLE, MARRIED, or UNKNOWN.
    • address(optional). Address of the employee. It is composed of the following elements:
      • street. Street of the address.
      • postalCode. Postal code of the address.
      • location. Location of the address.
      • latitude. Latitude of the address.
      • longitude. Latitude of the address.
    • emails (optional). List of emails used to contact the employee with.
    • phones (optional). List of phones used to contact the employee with.

    User

    • id (generated). Identifier of the user.
    • name (required). Name of the user.
      • Its length must be in the range [2..50].
      • Only alphabetic characters are allowed.
    • surname. Surname of the user.
    • code. High level identifier of the user used by the company. It is a number composed of 4 digits.
    • sex. Sex of the user. It can be MAN or WOMAN.
    • birthDate (optional). Birth date of the user.
    • civilStatus (optional). Civil status of the user. It can be SINGLE, MARRIED, or UNKNOWN.
    • address. Address of the user. It is composed of the following elements:
      • street. Street of the address.
      • postalCode. Postal code of the address.
      • location. Location of the address.
      • latitude. Latitude of the address.
      • longitude. Latitude of the address.
    • emails (optional). List of emails used to contact the employee with.
    • phones (optional). List of phones used to contact the employee with.
    • service. List of services required by the user. Each service is composed of the following elements:
      • description. Description of the service to do.
      • day. Day in which the service must be do. It can be MONDAY, TUESDAY,..., SUNDAY.
      • minutes. Duration time of the service, expressed in minutes.
      • startingTime. Starting time of the service, expressed in minutes.
      • endTime. Ending time of the service, expressed in minutes.

    It provides all the users. The returned users will be paginated to 25 items by default. Further pages can be requested by means of the page query paramenter. It can also set a custom page size up to 100 with the size parameter.

    HTTP request

    GET http://IP:PORT/VERSION/users

    Query parameters

    Parameter Description
    name Name of the user to find.
    page Zero-based page index. Default value: 0.
    size Size of the page to be returned. Default value: 25.

    Responses

    HTTP Code Description
    200 The request has succeeded and the users are included in the body of the response. The following headers are included in the response:
    • Pagination-Count: 1
    • Pagination-Size: 25
    • Pagination-Page: 0
    • Total-Count: 5
    where Pagination-Count is the number of existing pages, Pagination-Size is the size of each page, Pagination-Page is the index of the current page, and Total-Count is the number of entities in the server.
    404 The user with the requested name is not found.
