    Introduction

    REST API used to define, store, access, and execute algorithms. The algorithms are entities that receive a set of input parameter values and return a set of output parameter values. These algorithms can be machine learning methods, optimization techniques, or simulation models, among others.

    The API supports Cross Origin Resource Sharing (CORS) for AJAX requests from any origin. The following header is included in the corresponding responses: Access-Control-Allow-Origin: *. You can read the CORS W3C Recommendation to learn more about CORS.

    In the remainder of this documentation, it is assumed that the REST API is deployed in an IP and port denoted as IP and PORT, respectively.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "API_ENDPOINT"
  -H "Authorization: YOUR_API_KEY"

    Make sure to replace API_ENDPOINT with the corresponding API endpoint and YOUR_API_KEY with your API key.

    API requests are authenticated by a key which is provided by Kaizten. With every call to the API, the user must be authenticated. The API has a simple means to do this by passing in your key as an HTTP Authorization header with the request.

    When you signed up for Kaizten, a ‘default-key’ was generated for your convenience. This is the key that is used throughout the example code within algorithm pages. For these examples to work correctly, this default key must exist with all the permissions, otherwise the usage examples may result in a 401 Unauthorized error.

    You must replace YOUR_API_KEY with your personal API key.

    API versioning

    Kaizten Task REST API is versioned with the version specified on the URL route: IP:PORT/VERSION/API_ENDPOINT

    The current supported API version is v1 (i.e. IP:PORT/v1/API_ENDPOINT).

    Domain model

    Algorithm

    • id (generated). Identifier of the algorithm.
    • name (required). Name of the algorithm.
      • Its length must be in the range [2..100].
      • Digits, whitespaces, ., - and _ are valid characters.
    • tagline (optional). Short summary of the algorithm in only one line.
      • Its length must be in the range [1..100].
      • Digits, whitespaces, ., - and _ are valid characters.
    • description (optional). Description of the algorithm.
      • Its length must be in the range [1..1000].
      • Digits, whitespaces, ., - and _ are valid characters.
    • command (required). Command used to execute the algorithm.
      • Its length must be in the range [1..1000].

    Task

    • id. Identifier of the task.
    • name. Name of the task.
    • algorithmId. Identifier of the algorithm associated with the task. See Algorithm.
    • algorithmName. Name of the algorithm associated with the task.
    • startTime. Time in which the execution of the task starts. It is expressed in ISO 8601 format (e.g., 2020-12-31T13:32:23Z).
    • finishingTime. Time in which the execution of the task finishes. It is expressed in ISO 8601 format (e.g., 2020-12-31T13:32:23Z).
    • status. Status of the task when requested. See Task State.
    • exitCode. Status code to denote the termination of the task. The zero status is used when the execution is terminated successfully. However, a non-zero status code indicates an abnormal termination.
    • inputDataId. Identifier of the data used as input by the algorithm. See Data.
    • outputDataId. Identifier of the data provided as output by the algorithm. See Data.

    Task state

    • Pending. The task is accepted but has not started.
    • Canceled. The task is cancelled.
    • Running. The task is being executed.
    • Completed. The task is completed.

    In all the cases, the resource identifiers are generated by the system. This means that the client is not allowed to determine them.

    It provides all the algorithms. The returned algorithms will be paginated to 20 items by default. Further pages can be requested by means of the page query paramenter. It can also set a custom page size up to 100 with the size parameter. It can also provides a list of algorithms which fit the specified filters.

    HTTP Request

    GET http://IP:PORT/VERSION/algorithms

    Query Parameters

    Parameter Description
    name Name of the algorithm to find.
    page Zero-based page index. Default value: 0.
    size Size of the page to be returned. Default value: 20.
    filters It is a JSON string providing the data for the filters. This parameter is required, if it is not provided the call will respond with a Bad Request status.

    JASON filters Parameters

    Parameter Description
    name Regex used to filter the name of the algorithms
    labels Array of labels ids used for filtering algorithms by their labels. All the algorithms that have at least one of the provided labels will be returned.

    Responses

    HTTP Code Description
    200 The request has succeeded and the algorithms are included in the body of the response. The following headers are included in the response:
    • Pagination-Count: 1
    • Pagination-Size: 20
    • Pagination-Page: 0
    • Total-Count: 5
    where Pagination-Count is the number of existing pages, Pagination-Size is the size of each page, Pagination-Page is the index of the current page, and Total-Count is the number of entities in the server.
    404 The algorithm with the requested name is not found.

    Create a new algorithm

    	curl -X POST IP:PORT/VERSION/algorithms \
		-H "Accept: application/json" \
		-H "Content-Type: multipart/form-data" \ 
		-H "Authorization: YOUR_API_KEY" \
    -F "file=@FILE_TO_SEND" \
		-d 'form={ \
			"name": "vrp",
			"tagline": "Solver of the Vehicle Routing Problem",
			"description": "Variable Neighbourhood Search with 2 populations designed to create efficient delivery routes to customers",
			"command" : "java -cp vrp.jar com.kaizten.vrp.opt.main.Main %s"
		}'

    The above command returns a JSON structured like this:

    	{
		"id" : "Algorithm ID",
		"name" : "name-of-the-algorithm",
		"tagline": "Tagline of the algorithm.",
		"description": "Description of the algorithm.",
		"command" : "command",
		"fileName" : "file;yyyy/mm/dd-hh:mm:ss.extension",
		"createDate" : "yyyy/mm/dd hh:mm:ss",
		"lastUpdateDate" : "yyyy/mm/dd hh:mm:ss",
		"inputFileDescription" : "Description of the input file.",
		"outputFileDescription" : "Description of the output file.",
		"labels" : [
			{
				"id" : "Label ID.",
				"name" : "Label name."
			},
			{
				"id" : "Label ID.",
				"name" : "Label name."
			}
		]
	}

    It creates a new algorithm by specifying at least the required fields. The created algorithm is provided in the response.

    Only administrators can perform this operation.

    HTTP Request

    POST http://IP:PORT/VERSION/algorithms

    Request Parameters

    It can be provided the executable file of the algorithm as a parameter to the request, as follows.

    Parameter Description Required
    file The file of the executable. No

    Jason Form Parameters

    Inside the JASON specified in the body it can be provided the following parameters

    Parameter Description Required
    name The name of the algorithm. Yes
    description The description of the algorithm. No
    tagline The tagline of the algorithm. No
    command The command of the algorithm. No
    labels The labels of the algorithm. Must be specified as array of strings representing the names of the labels. No
    inputFileDescription The description of the input file of the algorithm. No
    outputFileDescription The description of the output file of the algorithm. No

    Responses

    HTTP Code Description
    201 The request has been fulfilled and has resulted in one algorithm being created. The new algorithm is included in the body of the response.
    400 There is an algorithm with the same name.

    Update an Algorithm

    	curl -X POST IP:PORT/VERSION/algorithms \
		-H "Accept: application/json" \
		-H "Content-Type: application/json" \ 
		-H "Authorization: YOUR_API_KEY" \
    -d "algorithmId=611ce6fc2658564692c77912" \
		-d '{ \
			"name": "vrp",
			"tagline": "Solver of the Vehicle Routing Problem",
			"description": "Variable Neighbourhood Search with 2 populations designed to create efficient delivery routes to customers",
			"command" : "java -cp vrp.jar com.kaizten.vrp.opt.main.Main %s"
		}'

    It update the data from an algorithm by specifying the fields to be modified.

    Only administrators can perform this operation.

    HTTP Request

    PUT http://IP:PORT/VERSION/algorithms

    Query Parameters

    Parameter Description
    algorithmId The ID of the algorithm to be modified.

    Parameters

    The parameters of the request are the exact same ones as in the create method.

    Responses

    HTTP Code Description
    200 The request modifications are successfully updated.
    404 There is no algorithm found, for the specified ID.

    Execute an Algorithm

    curl -X POST http://IP:PORT/VERSION/algorithms/:name \
    -H "Authorization: YOUR_API_KEY" \
    -H "Content-Type: text/json" \
    -d '{ 
            "taskName": "Schedule of users in SC",
            "inputDataId": "123456789" 
    }'

    The above command returns JSON structured like this:

    {
    "taskId" : "adfas1as1df56a11as",
    "taskName": "Schedule of users in SC",
    "algorithmId" : "5e6f565acff3233408180978",
    "algorithmName" : "solver-vehicle-routing-problem",
    "inputDataId": "123456789"
}

    It executes a specific algorithm by indicating its name. The algorithm can receive parameter values in the body of the request in the following formats:

    • Content-Type: application/json. JSON where each key is the name of the parameter and each value is the corresponding parameter value.

    The information about the task created is provided in the response.

    HTTP Request

    POST IP:PORT/VERSION/algorithms/:name

    URL Parameters

    Parameter Description
    name Name of the algorithm to execute.

    Responses

    HTTP Code Description
    202 The request has succeeded and the execution of the algorithm is queued. The information about the task created is provided in the response.
    404 The algorithm with the given name is not found.
