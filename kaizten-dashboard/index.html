    Introduction

    REST API used to manage dashboards in which variables in real time are displayed. In particular, the dashboards are composed of a set of widgets that represent the visual components that integrate them, whereas each widget allows to display variables according to its type. At the same time, the variables can be received as messages through Kafka to be registered.

    Domain model

    Dashboard

    • id. Identifier of the dashboard.
    • name. Name of the dashboard.
    • description. Description of the dashboard.
    • widgets. Array composed of the widgets included in the dashboard.

    Widget

    • id. Identifier of the widget.
    • name. Name of the widget.
    • description. Description of the widget.
    • x. X coordinate of the widget in the dashboard.
    • y. Y coordinate of the widget in the dashboard.
    • width. Width of the widget.
    • height. Height of the widget.
    • type. Type of widget. The values can be:
      • "VEGA_LITE". It indicates the content of the widget is a plot obtained through Vega-Lite. In this case, the widget has the following field:
        • jsonSpecification". It contains the JSON specification required by Vega-Lite to generate the corresponding plot. More information is available here.

    Variable

    • id. Identifier of the variable.
    • name. Name of the variable.
    • description. Description of the variable.
    • type. Type of the variable. The values can be:
      • "STRING".
        • allowedValues. Array composed of the potential values that the variable can has.
      • "LOGICAL".
      • "NUMERIC".
        • allowedValues. Array composed of the potential values that the variable can has.
        • minValue. Minimum value the variable can has.
        • maxValue. Maximum value the variable can has.
    • value. Value of the variable.
    • mappings. A comma-delimited list of key:value pairs. It is used to provide complementary properties in ad-hoc applications.

    Dashboards

    Get All Dashboards

    curl "http://localhost:8080/v1/dashboards"

    The above command returns JSON structured like this:

    PONER EJEMPLO

    It provides all the dashboards.

    HTTP Request

    GET http://localhost:8080/v1/dashboards

    Variables

    Get All Variables

    It provides all the variables.

    curl "http://localhost:8080/v1/variables"

    The above command returns JSON structured like this:

    [
   {
      "name" : "variable 0",
      "allowedValues" : [
         "VALUE 1",
         "VALUE 2",
         "VALUE 3"
      ],
      "type" : "STRING",
      "id" : "5d25f7c762cfb63d5874c3f3"
   },
   {
      "description" : "variable description",
      "name" : "variable 1",
      "allowedValues" : [
         "VALUE 1",
         "VALUE 2",
         "VALUE 3"
      ],
      "id" : "5d25f7c762cfb63d5874c3f4",
      "type" : "STRING"
   }
}

    HTTP Request

    GET http://localhost:8080/v1/variables

    Query Parameters

    Parameter Description
    name Name of the variable to find.
    type Type of variable. It can be STRING, LOGICAL, or NUMERIC.

    Responses

    HTTP Code Description
    200 The request has succeeded and the variables are included in the body of the response.
    400 The name or type of the variables to find is not valid.
    404 The variable with the requested name is not found.

    Create a New Variable

    It creates a new variable.

    curl -X POST -d '{"name":"customer name", "description":"Name of the customer", "type": "STRING"}' -H "Content-Type: application/json" "http://localhost:8080/v1/variables"

    The above command returns JSON structured like this:

    {
    "id":"5d25fd2e62cfb63d5874c3fd",
    "name":"customer name",
    "description":"name of the customer",
    "type":"STRING"
}

    HTTP Request

    POST http://localhost:8080/v1/variables

    Responses

    HTTP Code Description
    201 The request has been fulfilled and has resulted in one variable being created. The new variable is included in the body of the response.
    404 There is a variable with the same name.

    Get a Specific Variable

    It provides a specific variable.

    curl "http://localhost:8080/v1/variables/6e34df1e62cfb63d5874c394"

    The above command returns JSON structured like this:

    {
   "type" : "NUMERICAL",
   "description" : "distance between points",
   "name" : "distance",
   "id" : "6e34df1e62cfb63d5874c394"
}

    HTTP Request

    GET http://localhost:8080/v1/variables/:id

    URL Parameters

    Parameter Description
    id Identifier of the requested variable.

    Responses

    HTTP Code Description
    200 The request has succeeded. The variable is included in the body of the response.
    404 The variable with the given identifier is not found.

    Update a Specific Variable

    It updates a specific variable.

    curl -X PUT -d '{"name":"First name"}' -H "Content-Type: application/json" "http://localhost:8080/v1/variables"

    The above command returns JSON structured like this:

    {
    "id":"5d25fd2e62cfb63d5874c3fd",
    "name":"First name",
    "description":"name of the customer",
    "type":"STRING"
}

    HTTP Request

    PUT http://localhost:8080/v1/variables/:id

    URL Parameters

    Parameter Description
    id Identifier of the variable to update.

    Responses

    HTTP Code Description
    200 The request has succeeded. The variable is included in the body of the response.
    404 The variable with the given identifier is not found.

    Delete a Specific Variable

    It deletes a specific variable.

    curl -X DELETE "http://localhost:8080/v1/variables/6e34df1e62cfb63d5874c394"

    The above command returns JSON structured like this:

    {

   "description" : "distance between points",
   "name" : "distance",

    HTTP Request

    DELETE http://localhost:8080/v1/variables/:id

    URL Parameters

    Parameter Description
    id Identifier of the variable to delete.

    Responses

    HTTP Code Description
    201 The request has succeeded. The variable is included in the body of the response.
    404 The variable with the given identifier is not found.

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
