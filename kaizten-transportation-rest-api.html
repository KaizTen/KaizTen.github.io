    Introduction

    REST API used to define, store, access, transportation information objects.

    In the remainder of this documentation, it is assumed that the REST API is deployed in an IP and port denoted as IP and PORT, respectively.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "API_ENDPOINT"
  -H "Authorization: YOUR_API_KEY"

    Make sure to replace API_ENDPOINT with the corresponding API endpoint and YOUR_API_KEY with your API key.

    API requests are authenticated by a key which is provided by Kaizten. With every call to the API, the user must be authenticated. The API has a simple means to do this by passing in your key as an HTTP Authorization header with the request.

    When you signed up for Kaizten, a ‘default-key’ was generated for your convenience. This is the key that is used throughout the example code within algorithm pages. For these examples to work correctly, this default key must exist with all the permissions, otherwise the usage examples may result in a 401 Unauthorized error.

    You must replace YOUR_API_KEY with your personal API key.

    API versioning

    Kaizten Transportation REST API is versioned with the version specified on the URL route: IP:PORT/VERSION/API_ENDPOINT

    The current supported API version is v1 (i.e. IP:PORT/v1/API_ENDPOINT).

    Domain model

    Client

    • id (generated). Identifier of the client.
    • identityDocument (required). The document number that indentifies the client.
      • The identity document length must be in the range [2...100].
    • name (optional). The name of the client.
      • The name length must be in the range [2...1000].
    • surname (optional). The surname of the client.
      • The surname length must be in the range [2...1000].
    • phone (optional). The phone is composed by two variables:
      • phoneNumber (optional). The phone number of the client.
      • The phone number length must be in the range [9...12].
      • comments (optional). The comments regarding to the phone number.
      • The comments length must be in the range [2...1000].
    • address (optional). The address is composed by five variables:
      • street (optional). The street where the client lives in.
      • The street length must be in the range [2...100].
      • postalCode (optional). The postal code of the client home.
      • The postal code length must be exactly 5.
      • city (optional). The city where the client lives in.
      • The city length must be in the range [2...100].
      • latitude (optional). The coordinate of the latitude from the client home.
      • longitude (optional). The coordinate of the longitude from the client home.
    • comments (optional). The comments regarding to the client.
      • The comments length must be in the range [2...1000].
