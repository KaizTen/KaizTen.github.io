    Introduction

    REST API used to define, store, access, transportation information objects.

    In the remainder of this documentation, it is assumed that the REST API is deployed in an IP and port denoted as IP and PORT, respectively.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "API_ENDPOINT"
  -H "Authorization: YOUR_API_KEY"

    Make sure to replace API_ENDPOINT with the corresponding API endpoint and YOUR_API_KEY with your API key.

    API requests are authenticated by a key which is provided by Kaizten. With every call to the API, the user must be authenticated. The API has a simple means to do this by passing in your key as an HTTP Authorization header with the request.

    When you signed up for Kaizten, a ‘default-key’ was generated for your convenience. This is the key that is used throughout the example code within algorithm pages. For these examples to work correctly, this default key must exist with all the permissions, otherwise the usage examples may result in a 401 Unauthorized error.

    You must replace YOUR_API_KEY with your personal API key.

    API versioning

    Kaizten Transportation REST API is versioned with the version specified on the URL route: IP:PORT/VERSION/API_ENDPOINT

    The current supported API version is v1 (i.e. IP:PORT/v1/API_ENDPOINT).

    Domain model

    Client

    • id (generated). Identifier of the client.
    • identityDocument (required). The document number that indentifies the client.
      • The identity document length must be in the range [2...100].
    • name (optional). The name of the client.
      • The name length must be in the range [2...100].
    • surname (optional). The surname of the client.
      • The surname length must be in the range [2...100].
    • phone (optional). The phone is composed by two variables:
      • phoneNumber (optional). The phone number of the client.
        • The phone number length must be in the range [9...13].
      • comments (optional). The comments regarding to the phone number.
        • The comments length must be in the range [2...1000].
    • address (optional). The address is composed by five variables:
      • street (optional). The street where the client lives in.
        • The street length must be in the range [2...100].
      • postalCode (optional). The postal code of the client home.
        • The postal code length must be exactly 5.
      • city (optional). The city where the client lives in.
        • The city length must be in the range [2...100].
      • latitude (optional). The coordinate of the latitude from the client home.
        • The value must be numeric.
      • longitude (optional). The coordinate of the longitude from the client home.
        • The value must be numeric.
    • comments (optional). The comments regarding to the client.
      • The comments length must be in the range [2...1000].

    Employee

    • id (generated). Identifier of the employee.
    • identityDocument (required). The document number that indentifies the employee.
      • The identity document length must be in the range [2...100].
    • name (optional). The name of the employee.
      • The name length must be in the range [2...100].
    • surname (optional). The surname of the employee.
      • The surname length must be in the range [2...100].
    • type (optional). The type of employee.
      • Currently the options available are [DRIVER, STOREKEEPER, ADMINISTRATOR]
    • phone (optional). The phone is composed by two variables:
      • phoneNumber (optional). The phone number of the employee.
        • The phone number length must be in the range [9...13].
      • comments (optional). The comments regarding to the phone number.
        • The comments length must be in the range [2...1000].
    • email (optional). It is composed by two variables:
      • email (optional). The email of the employee.
        • The email length must be in the range [2...100].
      • comments (optional). The comments regarding to the email.
        • The comments length must be in the range [2...1000].
    • address (optional). The address is composed by five variables:
      • street (optional). The street where the employee lives in.
        • The street length must be in the range [2...100].
      • postalCode (optional). The postal code of the employee home.
        • The postal code length must be exactly 5.
      • city (optional). The city where the employee lives in.
        • The city length must be in the range [2...100].
      • latitude (optional). The coordinate of the latitude from the employee home.
        • The value must be numeric.
      • longitude (optional). The coordinate of the longitude from the employee home.
        • The value must be numeric.

    Product

    • id (generated). Identifier of the product.
    • name (optional). The name of the product.
      • The name length must be in the range [2...100].
    • description (optional). The description of the product.
      • The description length must be in the range [2...1000].
    • weight (optional). The weight of the product.
      • The value must be numeric greater than 0.
    • dimensions (optional). The dimensions of the product, which is split in 3 variables:
      • depth (optional). The depth of the product.
        • The value must be numeric greater than 0.
      • width (optional). The width of the product.
        • The value must be numeric greater than 0.
      • height (optional). The height of the product.
        • The value must be numeric greater than 0.

    Vehicle

    • id (generated). Identifier of the vehicle.
    • carPlate (required). The car plate of the vehicle.
      • The car plate length must be in the range [2...100].
    • buyYear (optional). The year when the vehicle was bought.
      • The value must be numeric greater than 1900 and smaller than 2100.
    • registeredYear (optional). The year when the vehicle was registered.
      • The value must be numeric greater than 1900 and smaller than 2100.
    • capacity (optional). The storage capacity of the vehicle.
      • The value must be numeric greater than 0. (Default value 0)
    • weight (optional). The weight of the vehicle.
      • The value must be numeric greater than 0. (Default value 0)
    • dimensions (optional). The dimensions of the vehicle, which is split in 3 variables:
      • depth (optional). The depth of the vehicle.
        • The value must be numeric greater than 0.
      • width (optional). The width of the vehicle.
        • The value must be numeric greater than 0.
      • height (optional). The height of the vehicle.
        • The value must be numeric greater than 0.

    Warehouse

    • id (generated). Identifier of the warehouse.
    • name (optional). The name of the warehouse.
      • The name length must be in the range [2...100].
    • capacity (optional). The storage capacity of the warehouse.
      • The value must be numeric greater than 0. (Default value 0)
    • address (optional). The address is composed by five variables:
      • street (optional). The street where the warehouse is located.
        • The street length must be in the range [2...100].
      • postalCode (optional). The postal code of the warehouse lacation.
        • The postal code length must be exactly 5.
      • city (optional). The city where the warehouse is located.
        • The city length must be in the range [2...100].
      • latitude (optional). The coordinate of the latitude from the warehouse.
        • The value must be numeric.
      • longitude (optional). The coordinate of the longitude from the warehouse.
        • The value must be numeric.
    • phone (optional). The phone is composed by two variables:
      • phoneNumber (optional). The phone number of the warehouse.
        • The phone number length must be in the range [9...13].
      • comments (optional). The comments regarding to the phone number.
        • The comments length must be in the range [2...1000].
    • productUnits (optional). The cuantity of the product units in the warehouse.
      • The value must be numeric greater than 0. (Default value 0)
    • vehicles (optional). A list of vehicles which are asociated with the warehouse.
    • employees (optional). A list of employees which are asociated with the warehouse.

  },

    "identityDocument": "131238244J",
    "name": "Sandra",
    "surname": "Jimenez García",
    "phone": {
      "phoneNumber": "652443935",
      "comments": "this is the main phone"

      "longitude":-16.32
    },
    "comments":"This client use to buy Bridgestone"
  }
]

    It provides all the clients registered. The returned clients will be paginated to 20 items by default. Further pages can be requested by means of the page query paramenter. It can also set a custom page size up to 100 with the size parameter.

    HTTP Request

    GET http://IP:PORT/VERSION/clients

    Query Parameters

    Parameter Description
    page Zero-based page index. Default value: 0. Min value: 0.
    size Size of the page to be returned. Default value: 20. Min value: 0. Max value: 100.

    Responses

    HTTP Code Description
    200 The request has succeeded and the clients are included in the body of the response. The following headers are included in the response:
    • Pagination-Count: 1
    • Pagination-Size: 20
    • Pagination-Page: 0
    • Total-Count: 5
    where Pagination-Count is the number of existing pages, Pagination-Size is the size of each page, Pagination-Page is the index of the current page, and Total-Count is the number of entities in the server.

    Create a client

    curl http://IP:PORT/VERSION/clients \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -H "Authorization: YOUR_API_KEY" \
  -d '{ \
    "identityDocument":"834237656874D", \
    "name":"Andres", \
    "surname": "Rosado Garcia"
  }'

    The above command returns JSON structured like this:

    {
  "address" : null,
  "comments" : null,
  "id" : "616d83379bbd37254785a7d6",
  "identityDocument" : "834237656874D",
  "name" : "Andres",
  "phone" : null,
  "surname" : "Rosado Garcia"
}

    It creates a new client by specifying at least the required fields. The created client is provided in the response.

    HTTP Request

    POST http://IP:PORT/VERSION/clients

    Body Parameters

    Paramenter Description Required
    identityDocument Identifier of the client document. Yes
    name Name of the client. No
    surname Surname of the client. No
    phone Phone information of the client. No
    address Address information of the client. No

    Phone Parameters

    Paramenter Description Required
    phoneNumber Phone number of the client. No
    comments Comments of the phone number. No

    Address Parameters

    Paramenter Description Required
    street Street of the client's home location. No
    postalCode Postal code of the client's home location. No
    city City of the client's home location. No
    latitude Latitude of the client's home location. No
    longitude Longitude of the client's home location. No

    Responses

    HTTP Code Description
    201 The client has been created successfully. The new client is included in the body of the response.
    400 Malformed body json.
