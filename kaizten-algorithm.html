    Introduction

    REST API used to define, store, access, and execute algorithms. The algorithms are entities that receive a set of input parameter values and return a set of output parameter values. These algorithms can be machine learning methods, optimization techniques, or simulation models, among others.

    In the remainder of this documentation, it is assumed that the REST API is deployed in an IP and port denoted as IP and PORT, respectively.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "API_ENDPOINT"
  -H "Authorization: YOUR_API_KEY"

    Make sure to replace API_ENDPOINT with the corresponding API endpoint and YOUR_API_KEY with your API key.

    API requests are authenticated by a key which is provided by Kaizten. With every call to the API, the user must be authenticated. The API has a simple means to do this by passing in your key as an HTTP “Authorization” header with the request.

    When you signed up for Kaizten, a ‘default-key’ was generated for your convenience. This is the key that is used throughout the example code within algorithm pages. For these examples to work correctly, this default key must exist with all the permissions, otherwise the usage examples may result in a 401 Unauthorized error.

    You must replace YOUR_API_KEY with your personal API key.

    API versioning

    Kaizten Algorithm API is versioned with the version specified on the URL route: IP:PORT/VERSION/API_ENDPOINT

    The current supported API version is v1 (i.e. IP:PORT/v1/API_ENDPOINT).

    Domain model

    Algorithm

    • id (generated). Identifier of the algorithm.
    • name (required). Name of the algorithm.
    • tagline (optional). Short summary of the algorithm in only one line.
    • description (optional). Description of the algorithm.
    • sampleInput (optional). Sample of the input used by the algorithm.

    Label

    • id (generated). Identifier of the label.
    • name (required). Name of the label. Its length must be in the range [2..25].

    Task

    • id. Identifier of the task.
    • name. Name of the task.
    • description. Description of the task.
    • userId. xxx
    • algorithmName. xxx
    • requestDate. xxx
    • startDate. xxx
    • finishDate. xxx
    • status. xxx

    Task states

    • Pending. La tarea aún no ha sido asignada a ningún worker para su realización.
    • Running. La tarea se está realizando.
    • Canceled. La tarea ha sido cancelada.
    • CompletedErrors. La tarea ha sido finalizada porque se han encontrado errores en la ejecución.
    • CompletedTimeout. La tarea ha sido finalizada porque se ha alcanzado el tiempo máximo de ejecución.
    • Completed. La tarea ha sido finalizada con éxito.

    In all the cases, the resource identifiers are generated by the system. This means that the client is not allowed to determine them.

    Labels

    Get all Labels

    curl http://IP:PORT/VERSION/labels \
    -H "Authorization: YOUR_API_KEY"

    The above command returns JSON structured like this:

    [
   {
      "id" : "5e6f563bcff3233408180977",
      "name" : "data-science"
   },
   {
      "id" : "5e6f565acff3233408180978",
      "name" : "optimization"
   },
   {
      "id" : "5e6f565acff3233408180979",
      "name" : "simulation"
   }
]

    It provides all the labels.

    HTTP Request

    GET http://IP:PORT/VERSION/labels

    Query Parameters

    Parameter Description
    name Name of the label to find.
    page Zero-based page index. Default value: 0.
    size Size of the page to be returned. Default value: 25.

    Responses

    HTTP Code Description
    200 The request has succeeded and the labels are included in the body of the response.
    404 The label with the requested name is not found.

    Create a new label

    curl -X POST IP:PORT/VERSION/labels \
    -H "Accept: application/json" 
    -H "Content-Type: application/json" 
    -H "Authorization: YOUR_API_KEY" 
    -d '{"name": "data-science"}'

    The above command returns JSON structured like this:

    {
   "id" : "5e6f565acff3233408180978",
   "name" : "data-science"
}

    It creates a new label by specifying its name.

    Only administrators can perform this operation.

    HTTP Request

    POST http://IP:PORT/VERSION/labels

    Responses

    HTTP Code Description
    201 The request has been fulfilled and has resulted in one label being created. The new label is included in the body of the response.
    404 There is a label with the same name.

    Get a Specific Label

    It provides a specific label by indicating its identifier.

    curl http://IP:PORT/VERSION/labels/:id \
    -H "Authorization: YOUR_API_KEY"

    The above command returns JSON structured like this:

    {
   "id" : "5e6f565acff3233408180978",
   "name" : "data-science"
}

    HTTP Request

    GET IP:PORT/VERSION/labels/:id

    URL Parameters

    Parameter Description
    id Identifier of the requested label.

    Responses

    HTTP Code Description
    200 The request has succeeded. The label is included in the body of the response.
    404 The label with the given identifier is not found.

    Update a Specific Label

    curl -X PUT http://IP:PORT/VERSION/labels/:id \
    -H "Content-Type: application/json" \
    -H "Authorization: YOUR_API_KEY"
    -d '{"name": "data-science2"}' -v | json_pp

    The above command returns JSON structured like this:

    {
   "id" : "5e6f565acff3233408180978",
   "name" : "data-science2"
}

    It updates a specific label by indicating its identifier.

    Only administrators can perform this operation.

    HTTP Request

    PUT http://IP:PORT/VERSION/labels/:id

    URL Parameters

    Parameter Description
    id Identifier of the label to update.

    Responses

    HTTP Code Description
    200 The request has succeeded. The label is included in the body of the response.
    404 The label with the given identifier is not found.

    Delete a Specific Label

    curl -X DELETE IP:PORT/VERSION/labels/:id 
    -H "Authorization: YOUR_API_KEY"

    The above command returns JSON structured like this:

    {
   "id" : "5e6f565acff3233408180978",
   "name" : "data-science"
}

    It deletes a label using its identifier (:id).

    Only administrators can perform this operation.

    HTTP Request

    DELETE http://IP:PORT/VERSION/labels

    Responses

    HTTP Code Description
    200 The request has been fulfilled and has resulted in one label being deleted. The deleted label is included in the body of the response.
    404 The label to delete does not exist.
