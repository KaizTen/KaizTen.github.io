    Introduction

    REST API used to define, store, access, and execute algorithms. The algorithms are entities that receives a set of input parameter values and return a set of output parameter values. These algorithms can be machine learning methods, optimization techniques, or simulation models, among others.

    Domain model

    Algorithm

    • id. Identifier of the algorithm
    • name. Name of the algorithm.
    • description. Description of the algorithm.
    • publishingDate. Date in which the algorithm was published in the platform.
    • lastUpdateDate. Last date in which the algorithm was updated.
    • inputParameters. List of input parameters.
    • outputParameters. List of output parameters.
    • images. List composed of the URLs of the images that represent the algorithm.
    • owners. List of user identifiers that own the algorithm.
    • allowedUsers. List of users not included in the owners list that can be execute the algorithm.
    • labels. List of labels associated with the algorithm.

    Labels

    Get all Labels

    It provides all the labels.

    curl "http://localhost:8080/v1/variables"

    The above command returns JSON structured like this:

    [
   {
      "name" : "variable 0",
      "allowedValues" : [
         "VALUE 1",
         "VALUE 2",
         "VALUE 3"
      ],
      "type" : "STRING",
      "id" : "5d25f7c762cfb63d5874c3f3"
   },
   {
      "description" : "variable description",
      "name" : "variable 1",
      "allowedValues" : [
         "VALUE 1",
         "VALUE 2",
         "VALUE 3"
      ],
      "id" : "5d25f7c762cfb63d5874c3f4",
      "type" : "STRING"
   }
}

    HTTP Request

    GET http://localhost:8080/v1/variables

    Query Parameters

    Parameter Description
    name Name of the variable to find.
    type Type of variable. It can be STRING, LOGICAL, or NUMERIC.

    Responses

    HTTP Code Description
    200 The request has succeeded and the variables are included in the body of the response.
    400 The name or type of the variables to find is not valid.
    404 The variable with the requested name is not found.

    Create a New Label

    It creates a new label.

    curl -X POST -d '{"name":"customer name", "description":"Name of the customer", "type": "STRING"}' -H "Content-Type: application/json" "http://localhost:8080/v1/variables"

    The above command returns JSON structured like this:

    {
    "id":"5d25fd2e62cfb63d5874c3fd",
    "name":"customer name",
    "description":"name of the customer",
    "type":"STRING"
}

    HTTP Request

    POST http://localhost:8080/v1/variables

    Responses

    HTTP Code Description
    201 The request has been fulfilled and has resulted in one variable being created. The new variable is included in the body of the response.
    404 There is a variable with the same name.

    Get a Specific Label

    It provides a specific label.

    curl "http://localhost:8080/v1/variables/6e34df1e62cfb63d5874c394"

    The above command returns JSON structured like this:

    {
   "type" : "NUMERICAL",
   "description" : "distance between points",
   "name" : "distance",
   "id" : "6e34df1e62cfb63d5874c394"
}

    HTTP Request

    GET http://localhost:8080/v1/variables/:id

    URL Parameters

    Parameter Description
    id Identifier of the requested variable.

    Responses

    HTTP Code Description
    200 The request has succeeded. The variable is included in the body of the response.
    404 The variable with the given identifier is not found.

    Update a Specific Label

    It updates a specific label.

    curl -X PUT -d '{"name":"First name"}' -H "Content-Type: application/json" "http://localhost:8080/v1/variables"

    The above command returns JSON structured like this:

    {
    "id":"5d25fd2e62cfb63d5874c3fd",
    "name":"First name",
    "description":"name of the customer",
    "type":"STRING"
}

    HTTP Request

    PUT http://localhost:8080/v1/variables/:id

    URL Parameters

    Parameter Description
    id Identifier of the variable to update.

    Responses

    HTTP Code Description
    200 The request has succeeded. The variable is included in the body of the response.
    404 The variable with the given identifier is not found.

    Delete a Specific Label

    The above command returns JSON structured like this:

   "description" : "distance between points",
   "name" : "distance",
   "id" : "6e34df1e62cfb63d5874c394"
}

    URL Parameters

    Responses

    HTTP Code Description
    201 The request has succeeded. The variable is included in the body of the response.
    404 The variable with the given identifier is not found.

    Authentication

    To authorize, use this code:

    require 'kittn'

api = Kittn::APIClient.authorize!('meowmeowmeow')

    import kittn

api = kittn.authorize('meowmeowmeow')

    # With shell, you can just pass the correct header with each request
curl "api_endpoint_here"
  -H "Authorization: meowmeowmeow"

    const kittn = require('kittn');

let api = kittn.authorize('meowmeowmeow');

    Make sure to replace meowmeowmeow with your API key.

    Kittn uses API keys to allow access to the API. You can register a new Kittn API key at our developer portal.

    Kittn expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: meowmeowmeow

    You must replace meowmeowmeow with your personal API key.

    Kittens

    Get All Kittens

    require 'kittn'

api = Kittn::APIClient.authorize!('meowmeowmeow')
api.kittens.get

    import kittn

api = kittn.authorize('meowmeowmeow')
api.kittens.get()

    curl "http://example.com/api/kittens"
  -H "Authorization: meowmeowmeow"

    const kittn = require('kittn');

let api = kittn.authorize('meowmeowmeow');
let kittens = api.kittens.get();

    The above command returns JSON structured like this:

    [
  {
    "id": 1,
    "name": "Fluffums",
    "breed": "calico",
    "fluffiness": 6,
    "cuteness": 7
  },

    "name": "Max",

    This endpoint retrieves all kittens.

    HTTP Request

    GET http://example.com/api/kittens

    Query Parameters

    Parameter Default Description
    include_cats false If set to true, the result will also include cats.
    available true If set to false, the result will include kittens that have already been adopted.
    Remember — a happy kitten is an authenticated kitten!

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
